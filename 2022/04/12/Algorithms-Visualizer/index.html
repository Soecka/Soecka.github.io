分支操作

继续本地环境配置
1. 安装Node 12

	

2. 查询文档的时候意外收获，有[博主教程](https://blog.csdn.net/luoqiang616477607/article/details/122563583)建议使用 Python2 ，其他内容语焉不详（这也是小白写这篇博客的出发点 :v: ），于是在本地继续配置 Python2 ，继续查教程，按照[How to Switch Python Version in Ubuntu & Debian](https://tecadmin.net/how-to-switch-python-version-in-ubuntu-debian/)教程步骤，切换 Python

	

3. npm报错太多，换yarn（继续使用npm也没影响，个人喜好）
	

4. 成功安装，但是此时未能实现算法可视化，按照文档说明[algorithm-visualizer](https://github.com/algorithm-visualizer/algorithm-visualizer)/**[CONTRIBUTING.md](https://github.com/algorithm-visualizer/algorithm-visualizer/blob/master/CONTRIBUTING.md)**，打开`package.json` , 进行如下操作，一切正常
	
	
5. 现在才发现官方项目文档下面说明有Gitoid链接，开启第二分支操作，尝试使用 Gitpod 配置项目，（到这里就已经结束了，下面是自由发挥）

2. 项目仍是报错，按上述方法继续配置node版本

3. 报错 “ Invalid Host Header”, 在这一点上僵持了很久，我以为还是Python的问题，切换至Python也无变化

4. 继续查询，Gitpod 官网找到了解决办法 [Invalid Host Header](https://www.gitpod.io/blog/gitpodify/#invalid-host-header)

   

5. 所以先要 ‘export DANGEROUSLY_DISABLE_HOST_CHECK=true’，然后再yarn install，自己fork项目测试一下，修改  `.gitpod.yml`

   

6. 成功了，弱弱的创建一个pull request，帮助像我一样的无知小白
